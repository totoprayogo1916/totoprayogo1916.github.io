Mengembangkan MBlog

Penulis
  • Author
    Toto

Pernah ngeblog, tapi sayangnya selalu berhenti ditengah jalan seakan gak ada semangatnya untuk menulis blog. Mulai dari menggunakan Blogger, Wordpress dan berbagai macamnya. Semua berhenti karena hanya ingin mencoba fiturnya.

Intro #

Lagi seneng-senengnya ngoding mungkin jadi alasan untuk gak semangat mencari materi untuk ngeblog. Sampai urusan pekerjaan juga menghambat hal serupa. Pernah mencoba membuat sistem blog sendiri dengan koding sendiri. Baru beberapa langkah sudah tidak ada inspirasi untuk melanjutkan ngodingnya, maklum cuma diwaktu selingan saja dapat mengerjakannya.

Dan dalam topik ini, MBlog pernah dan sempat untuk dikembangkan sendiri sesuai kebutuhan. Oh iya, disini (mfikri.com) saya mendapatkan source Mblog.

Pengembangan MBlog #

Keduanya baik MBlog v1 & v2; saya rasakan hanya berubah dari segi tampilan saja. Tapi tidak ada masalah yang penting jalan. Keduanya menggunakan PHP framework CodeIgniter3 dengan bawaan struktur folder dari CI.

Dari keduanya saya telah mengubahnya menggunakan dengan struktur folder yang lebih aman. Sama seperti Laravel, CodeIgniter4 dan mungkin yang lainnya yang telah menerapkan struktur folder yang lebih aman. Dimana root folder yang digunakan telah dipisahkan dari sistem (core) maupun public.

Tambahan lagi dari keduannya MBlog v1 & v2 juga sudah menggunakan composer untuk menginstall core nya. Sehingga mudah untuk upgrade sistem (core) maupun menambahkan pluginnya. Dari kedua versi tersebut saya memberi label MBlog v1.5 dan v2.5 sebagai pembeda versi pengembangan.

Dan kini juga sudah saya kembangkan dengan meningkatkan penggunaan CodeIgniter4 sebagai core nya. Yang lainnya adalah merubah url postnya, dari sebelumnya

http://example.com/blog/4-cara-menjadi-lebih-aktif-dalam-belajar

kini saya rubah menjadi seperti berikut;

http://example.com/blog/12
http://example.com/blog/12/ini-asal
http://example.com/blog/12/4-cara-menjadi-lebih-aktif-dalam-belajar

Penjelasannya sebagai berikut; untuk baris pertama dapat digunakan sebagai link pendek yang akan redirect otomatis ke contoh baris ketiga (yang sebenarnya) dan untuk url contoh baris kedua (dengan slug lama) juga akan dikirim ke contoh format baris ketiga. Jadi url yang dihasilkan akan selalu baru meski judulnya berubah. Tidak ada modifikasi slug pada post untuk model clickbait.

Saat ini sudah dapat digunakan, tapi tetap saya test untuk mengetahui semua fungsinya berjalan minimal seperti pada sebelumnya. Kemungkinan akan saya gunakan untuk menulis blog dikemudian hari.
